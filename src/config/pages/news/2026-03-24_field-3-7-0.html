New features

  • Support for upcoming QGIS plugin "Field Connect.
  • New option in field editor for selecting allowed geometry types for the geometry field of a category.
  • New context menu in warnings menu providing the options "View in context" and "Edit"
  • New REST API endpoints
    • POST /fileImport (Import images and world files)
    • GET /fileExport (Export images and world files)

Changes

  • QR codes can now be enabled for resources of category "Type"
  • Types can now be linked to find resources by scanning the QR code via the context menu option "Scan type" (if QR codes are also enabled for storage places, the option is called "Scan type or storage place")
  • The semicolon character is not allowed anymore in the field "Identifier" and in the fields "Staff" and "Campaigns" of the project properties
  • The semicolon character cannot be used anymore when entering the identifier of a new value of a valuelist
  • When exporting fields of input type "Single line text (List)" to CSV, one column is now always created for each list entry (before, this was only true for fields with input in multiple languages). Importing CSV files that contain fields of this input type with entries separated by a semicolon will no longer work.
  • Improved synchronization performance for newly downloaded projects
  • The read-only field "Original filename" is now displayed for image resources and included in exported CSV files
  • When restoring a backup file, the restored project is now opened immediately
  • The REST API endpoint "/import" now allows adding data to import processes across multiple POST requests via the new parameters "command" and "importId"
  • An error message is now shown when trying to import a QR code that is already used for another resource
  • A new warning is now shown for duplicate QR codes
  • Warnings can now be resolved by editing the affected resources via import (please note that the import will still fail if the warnings are not resolved by the import)
  • When importing data via REST API, the resources view is now updated immediately to show newly imported resources
  • When validating fields of input type "Dating", it is now checked that each entry contains only allowed subfields
